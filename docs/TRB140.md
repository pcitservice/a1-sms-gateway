# Teltonika TRB140 integration

The TRB140 runs **RutOS**, which exposes a JSON HTTP API documented at
<https://wiki.teltonika-networks.com/view/RutOS_API>. Our driver uses that
API for everything it can, and falls back to SSH + raw AT commands for
operations the API doesn't cover.

## Authentication

`POST /api/login`

```json
{ "username": "admin", "password": "..." }
```

Returns `{ "ubus_rpc_session": "<token>" }`. We cache the token in Redis
keyed by `gateway:{id}:token` with a 5-minute TTL (the device's session is
~10 min by default). On 401, the driver re-authenticates once before
surfacing the error.

## Sending an SMS

`POST /api/messages/actions/send`

```json
{ "data": { "number": "+4512345678", "message": "Hello", "modem": "2-1" } }
```

Successful response:

```json
{ "success": true, "data": { "sms_used": 1 } }
```

The TRB140 does not return a provider message ID on send, so the driver
synthesises a deterministic ID by hashing `(modem, timestamp_ms, dst,
fingerprint)`.

## Receiving SMS

`GET /api/messages/inbox?modem=2-1`

Returns an array of messages with `id`, `sender`, `text`, `date`. After
ingesting, the driver calls
`POST /api/messages/actions/delete` with the IDs to clear them from the
device — otherwise the SIM storage fills up and the device starts dropping
inbound.

## Device health

| Need               | Endpoint                       |
| ------------------ | ------------------------------ |
| Signal RSSI / RSRP | `GET /api/modem/status`        |
| Operator + tech    | `GET /api/modem/connection`    |
| SIM status         | `GET /api/sim/status`          |
| Uptime / load      | `GET /api/system/info`         |
| LTE info           | `GET /api/modem/lte`           |
| Reboot             | `POST /api/system/actions/reboot` |

## Multi-SIM

TRB140 has a single SIM slot but newer firmware on TRB145 / RUT240 exposes
`modem` IDs like `2-1`, `3-1`. The driver treats the `modem` field as an
opaque routing key — `gateways.modem_id` selects which one to use.

## SSH fallback

For commands not in the API (e.g. `at!gstatus?`, custom `gsmctl` invocations),
the driver SSHes in as `root` with the key at `TRB140_SSH_KEY_PATH` and pipes
through `gsmctl -A "AT+CMD"`. Output is parsed line-by-line.

```php
$ssh->exec('gsmctl -A "AT+CSQ"');  // → +CSQ: 22,99
```

## Watchdog

A scheduled command (`a1:gateway:watchdog`) runs every minute. It calls
`health()` on every active gateway. If three consecutive failures occur,
the gateway is marked `offline`, a `GatewayOffline` event fires (which
triggers a webhook to the team owner), and the manager stops routing SMS
to it. The next successful `health()` re-marks it `online` and emits
`GatewayRecovered`.

## Auto reconnect

When health reports `connection_state != connected`, the driver issues a
`POST /api/modem/actions/reconnect`. If three reconnect attempts in 10
minutes all fail, it escalates to `POST /api/system/actions/reboot`.

## Configuration knobs (per gateway row)

| Column            | Notes                                                                 |
| ----------------- | --------------------------------------------------------------------- |
| `kind`            | `trb140`, `huawei`, `mock`                                            |
| `host`            | IP/hostname                                                           |
| `port`            | HTTP port (80/443)                                                    |
| `protocol`        | `http` / `https`                                                      |
| `username`        | RutOS username                                                        |
| `password`        | encrypted at rest (Laravel `encrypted` cast)                          |
| `modem_id`        | which modem on multi-modem devices                                    |
| `ssh_enabled`     | bool                                                                  |
| `ssh_key_ref`     | reference to a secret managed by the platform                         |
| `rate_limit_smin` | max SMS per minute (TRB140 ≈ 6 with consumer SIMs)                    |
| `daily_cap`       | optional cap to prevent SIM-card carrier bans                         |
| `team_id`         | tenant that owns this gateway (NULL = platform pool / pay-per-use)    |
